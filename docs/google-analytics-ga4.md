# Google Analytics 4(GA4) 설정 가이드 (themoa.me / Next.js)

이 프로젝트는 `app/layout.tsx`에서 `@next/third-parties/google`의 `GoogleAnalytics`로 GA4를 불러옵니다.  
환경 변수 **`NEXT_PUBLIC_GA_MEASUREMENT_ID`**(형식: `G-xxxxxxxxxx`)만 있으면 동작합니다.

---

## 1. Google 계정 준비

- Google에 로그인할 수 있는 계정이 있으면 됩니다 (Gmail 등).
- 회사/개인 사이트라면 **소유·운영에 쓸 계정**으로 통일하는 것을 권장합니다.

---

## 2. Google Analytics에서 속성 만들기

1. 브라우저에서 [Google Analytics](https://analytics.google.com/) 접속 후 로그인합니다.
2. 처음이면 **측정 시작** 마법사가 나옵니다. 이미 쓰는 계정이 있으면 **관리**(톱니바퀴)로 이동합니다.
3. **계정 만들기** (또는 기존 계정 선택)
   - 계정 이름: 예) `themoa` (임의로 지어도 됨)
   - 데이터 공유 설정은 원하는 대로 선택 후 **다음**
4. **속성 만들기**
   - 속성 이름: 예) `themoa.me`
   - 보고 시간대: **대한민국** 권장
   - 통화: **KRW(원)** 등 실제에 맞게 선택
   - **다음**
5. **비즈니스 세부정보** (업종·규모 등)는 선택 사항입니다. 건너뛰거나 간단히 입력 후 진행합니다.
6. **약관 동의**가 나오면 읽고 동의합니다.

---

## 3. 데이터 스트림(웹) 연결 — 측정 ID 발급

1. **관리**(왼쪽 하단 톱니) → **데이터 스트림** (또는 설정 마법사에서) **웹** 선택.
2. **웹 스트림 추가**
   - **웹사이트 URL**: 실제 도메인 (예: `https://themoa.me`)  
     - 로컬만 테스트할 때는 나중에 스트림에 도메인을 추가하거나, 실서비스 URL 기준으로 만듭니다.
   - **스트림 이름**: 예) `themoa.me Web`
3. **스트림 만들기** 후 상세 화면에서 **측정 ID**를 확인합니다.
   - 형식: **`G-`로 시작하는 문자열** (예: `G-AB12CD34EF`)

이 값이 코드에서 쓰는 **`NEXT_PUBLIC_GA_MEASUREMENT_ID`** 입니다.

---

## 4. 로컬 프로젝트에 환경 변수 넣기

1. 저장소 루트에 **`.env.local`** 파일을 엽니다 (없으면 새로 만듭니다).
2. 아래 한 줄을 추가합니다 (`G-...`는 본인 측정 ID로 교체).

```env
NEXT_PUBLIC_GA_MEASUREMENT_ID=G-여기에_측정ID_붙여넣기
```

3. **저장**합니다.
4. 이미 `npm run dev`를 켜 두었다면 **개발 서버를 한 번 재시작**합니다.  
   (`NEXT_PUBLIC_*`는 빌드/시작 시점에 읽히는 경우가 많습니다.)

> **보안:** `.env.local`은 `.gitignore`에 포함되어 있어야 하며, **GitHub에 올리지 마세요.**  
> 측정 ID는 공개되어도 남이 임의로 사이트에 심는 정도의 위험은 있으므로, 저장소에 커밋하지 않는 것이 좋습니다.

---

## 5. 동작 확인

1. 브라우저에서 로컬 사이트 접속 (예: `http://localhost:3000`).
2. GA 화면에서 **보고서** → **실시간** (또는 **실시간 개요**)을 엽니다.
3. 같은 브라우저에서 사이트를 몇 페이지 이동해 봅니다.
4. 수 초~1~2분 안에 **활성 사용자** 또는 **이벤트**가 보이면 연결된 것입니다.

**개발자 도구로 스크립트 확인(선택):**

- F12 → **Network** 탭에서 `google-analytics.com` 또는 `googletagmanager.com` 요청이 있는지 확인합니다.
- **Application** → **Cookies**에 `_ga` 관련 쿠키가 생기는지 볼 수 있습니다.

---

## 6. 배포 환경(Vercel 등) 설정

로컬이 아닌 **실제 도메인**으로 서비스할 때:

1. 호스팅 대시보드(Vercel이면 **Project → Settings → Environment Variables**)로 이동합니다.
2. 변수 이름: **`NEXT_PUBLIC_GA_MEASUREMENT_ID`**
3. 값: GA에서 발급한 **`G-...`** (로컬과 동일해도 되고, 스트림을 나눴다면 배포용 ID를 넣습니다.)
4. **Production**(및 필요 시 Preview)에 적용 후 **재배포**합니다.

배포 URL이 GA 스트림에 등록한 도메인과 일치하는지 확인합니다.  
(필요하면 GA **데이터 스트림** 설정에서 허용 도메인을 점검합니다.)

---

## 7. 자주 있는 문제

| 증상 | 확인할 것 |
|------|-----------|
| 실시간에 안 잡힘 | 측정 ID 오타, `.env.local` 저장 여부, **dev 서버 재시작**, 광고 차단 확장 끄기 |
| 배포만 안 잡힘 | 호스팅 환경 변수에 `NEXT_PUBLIC_GA_MEASUREMENT_ID` 등록 여부, 재배포 여부 |
| 여전히 없음 | GA 속성이 GA4인지, 스트림이 **웹**인지, 브라우저 시크릿에서 한 번 테스트 |

---

## 8. 나중에 애드센스와 같이 쓸 때

- GA4와 애드센스는 **별도 승인·별도 스크립트**입니다. GA만 먼저 붙여 두어도 애드센스 승인 후 광고 코드를 추가하면 됩니다.
- 동의 배너(쿠키)는 **서비스 대상 국가·정책**에 따라 검토하세요.

---

## 9. 이 레포에서 코드가 있는 위치

- `package.json`: `@next/third-parties` 의존성
- `app/layout.tsx`: `GoogleAnalytics` 컴포넌트, `NEXT_PUBLIC_GA_MEASUREMENT_ID` 읽기

추가 이벤트(버튼 클릭 등)를 보내려면 `@next/third-parties/google`의 `sendGAEvent` 사용을 검토하면 됩니다.
